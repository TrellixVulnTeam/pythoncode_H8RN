用户使用手册

















 
HOLA 云盘是一款安装在用户台式或笔记本电脑中的客户端程序，它可以帮助用户完
成以下操作：

	访问您本地计算机中的所有HOLA文档 - HOLA云盘是一个Widnows应用，能够
很容易的访问HOLA系统中的所有文件。HOLA云盘也提供所有与文档或文件夹相
关的主要功能（上载、下载、删除、CURL，等）。

	将本地计算机文件备份至HOLA服务器 - 使用HOLA 云盘，用户无需通过浏览器
登录HOLA系统即可将文件轻松上传至HOLA服务器。
　　
	实现本地计算机和HOLA服务器之间的文件同步 - HOLA 云盘可以将所选择的文
件在本地计算机和HOLA服务器之间进行同步。同步虽然简单易用，但具备强大的
企业级管理特性。

1.安装HOLA 云盘
HOLA 云盘支持32位和64位操作系统，可以在以下操作系统上运行：

	Windows 8.1
	Windows 8
	Windows 7
	Windows Vista
	Windows XP

为了保证HOLA 云盘的正常使用，需要满足以下几点要求：

	服务器上已安装HOLA 系统 v6.2.3 或更高版本（企业版或标准版）
	HOLA Web Service已成功发布
	HOLA的产品使用许可中包含HOLA 云盘的授权
	HOLA管理员在HOLA管理端许可授权中已添加了可被授权使用HOLA 云盘的用
户

如果您还没有安装HOLA系统，或不知道如何发布Web Service，请阅读HOLA安装
手册。

如果您不知道如何授权用户使用HOLA 云盘，请阅读HOLA系统管理员手册-站点管理

3.2文档相关功能
访问文档相关的功能，选定某个文档，在按钮区域点击任意按钮。也可以右击选定的文
档，弹出与选定文档相关的菜单列表。

•	删除 – 可以从HOLA系统中删除选定文档。
•	分享 – 点击分享按钮，弹出窗口，用户可以复制文档的CURL，通过邮件、聊
天工具、或者其它的工具分享给HOLA中的其他用户。

4.1 备份 - 上传文件夹至HOLA服务器
正确安装和设置HOLA 云盘后（第2章 - HOLA 云盘初始设置），从本地计算机备份文

选择想要上传至HOLA服务器的文件夹并右击，选择“上载到HOLA”选项。

在窗口中选择HOLA工作区和指定文件夹位置。

4.	选择了上传文件的位置后，点击“确定”按钮以执行上传任务。也可以点击“取消”
关闭弹窗并取消上传。

5.	在文件上传过程中，右下方托盘中的HOLA云盘图标是旋转状态。旋转结束表示已

HOLA服务器获取更新，第4.6章 – 双向同步 – 同步设置 ，第4.7章 – 自动同步。

4.3同步 - 文件映射
除了使用HOLA 云盘将文件从本地计算机备份到HOLA服务器（一次性上传）以外，
还可以使用HOLA 云盘来同步文件，但是必须先创建您的本地计算机和HOLA服务器
之间的关联关系。

HOLA系统可以在两个文件夹（一个位于本地计算机，一个位于HOLA服务器）之间创
建同步链接。创建链接关系（映射），须进行以下操作：

1.	在本地计算机上选择想要同步至HOLA服务器的文件夹，右击并选择“映射到HOLA”
选项。


 

选择想要同步至HOLA服务器的文件夹，右击并选择“映射到HOLA”选项。

2.	系统自动弹出窗口，选择在HOLA服务器系统中被映射到的文件夹，点击“确定”
即可。
 

 

选择在HOLA服务器系统中被映射到的文件夹，点击“确定”即可。

3.	窗口自动弹出并询问是否立即同步内容或稍后进行。如果想马上同步，点击“是”
执行同步操作。如果不想立即执行同步操作，但仍旧想要创建同步链接，点击“否”。


 

如果想马上同步，点击“是”执行同步操作。如果不想立即执行同步操作，但仍旧想要创建同步链接，点击“否”。

4.	无论选择哪个选项，HOLA 云盘都将创建本地文件夹和HOLA服务器文件夹之间的
映射关系。该被映射文件夹的所有信息(包括所含文件、子文件夹)都会被映射到
HOLA服务器。映射文件夹或文件可以通过图标识别映射状态。文件夹可以有2种
映射状态，而文档可以有3种映射状态:

 
绿色图标文件夹-表示该文件夹已被映射至HOLA服务器，文件夹里的内容也已被同步。

 
橙色图标文件夹-表示该文件夹已被映射至HOLA服务器，但至少有一个文件（或子文件夹）未被同步到
HOLA服务器，或在同步过程中至少有一个文件发生了冲突或有其他问题。

 
绿色图标文件-表示该文件已被映射至HOLA服务器，并且已被同步至HOLA服务器。

4.8同步 - 同步冲突解决方法
HOLA 云盘能够解决在团队合作创建文档过程中可能产生冲突的同步问题。

在上次同步后，如果本地文件和服务器文件都被修改过，当HOLA 云盘再次进行同步
操作的时候（从HOLA服务器同步或上载到HOLA服务器），将发生同步冲突。

此时云盘无法确认哪份文件应作为最终版，因此HOLA 云盘会检查云盘设置并基于云
盘用户设置执行操作。

用户也可以在同步冲突发生时设置云盘如何操作。

从HOLA服务器更新时的冲突解决办法

在上次同步后，本地文件和服务器文件都曾被修改过，当云盘执行从HOLA服务器更新
操作时，云盘会检查应该如何处理本地文件。有两种选项可供选择：

	保留 - 本地文件会被保留（未同步）
	覆盖 - 本地文件会被服务器文件覆盖（同步）

提交到HOLA服务器时的冲突解决方法

在上次同步后，本地文件和服务器文件都曾被修改过，当云盘执行提交到HOLA服务器
的操作时， 云盘会检查应该如何处理服务器文件。有两种选项可供选择：

	保留 - 服务器文件会被保留（未同步）
	覆盖 - 服务器文件会被本地文件覆盖为新版本（同步）

同步冲突的解决方法

“同步”操作是“从HOLA更新”和“上载到HOLA”两项操作的结合，同步操作的冲突解决
方法也建立在两项操作的冲突解决方法之上。

设置方法

设置同步冲突，须进行如下操作：

1.	右击右下方托盘中的HOLA 云盘图标，选择“设置”选项。
 

 

右击右下方托盘中的HOLA 云盘图标，选择“设置”选项。

2.	在设置弹窗中点击“冲突设置”按钮。


 

同步冲突设置可以在设置弹窗中点击“冲突设置”按钮进行。

3.	为“从HOLA更新”和“提交到HOLA”设置冲突解决方法。选择合适的选项，点击“确
认”保存设置，或点击“取消”不保存任何设置。


 

为“从HOLA更新”和“提交到HOLA”选择设置冲突解决方法。

现在可以保存新的冲突解决方法设置，设置会立即生效并作用于“从HOLA更新”和“提交
到HOLA”同步操作。

7. 附录A - 错误（红色）文件图标的原因
以下列表为可能造成文件上出现错误（红色）图标的状况。

1.	提交至HOLA - 同名文
件已存在于服务器中。
1.	已映射文件夹至HOLA，但还没有进行同步步骤。
2.	映射的本地文件夹存在一个a.doc文件。
3.	有人上传了一份新文件至HOLA 服务器的对应的映射
文件夹中，该文件名也为a.doc（但内容不同）。
4.	在HOLA服务其中已经存在在相同文件名的文件
5.	“提交至HOLA”操作失败。
2.	提交至HOLA - 安全权
限设置有改动，不允许
提交任何文件。
1.	用户已经将本地文件夹映射和同步到HOLA.
2.	在映射的本地文件夹中对其中一个文件进行过修改。
3.	有人在服务器上改变了对应文件的安全权限（因此用
户无法再上传新版文件）。
4.	没有权限上传任何新版文件。
5.	“提交至HOLA”操作失败。
3.	从HOLA获取更新文件 
- 同名文件已经存在于
本地计算机。
1.	用户已经将本地文件夹映射和同步至HOLA.
2.	有人在服务器中上传了新文件，文件名为a.doc。
3.	您在本地计算机中创建了新文件（在已映射文件夹
中），命名为a.doc,但内容与服务器上的同名文件不同。
4.	本地计算机中已经存在同名文件。
5.	“从HOLA获取更新文件”操作失败。
4.	从HOLA获取更新文件 
- 文件已从服务器删
除。
1.	用户已将本地文件夹映射和同步至服务器。
2.	有人在服务器中删除了被同步文件。
3.	应该更新的文件已将从服务器中被删除。
4.	“从HOLA获取更新文件”操作失败。
5.	从HOLA获取更新文件 
- 安全权限设置中有改
动，不允许更新任何文
件。
1.	您已将您的本地文件夹映射和同步至服务器。
2.	有人在服务器中对文件改变了安全权限（因此不能下
载文件）。
3.	有人在服务器中上传了新版文件。
4.	没有安全权限去下载任何服务器中的文件。
5.	“从HOLA获取更新文件”运行失败。
6.	映射到HOLA – 在映射
过程中文件名有冲突
1.	在本地一个文件夹（文件夹01）里存在一个a.doc文
件。
2.	在服务器上的一个文件夹（文件夹02）里存在一个同
名为a.doc的文件，但是该文件的内容是不相同的。
3.	如果您将这两个文件夹（文件夹01和文件夹02）建
立映射关系，由于文件夹中的文件有同名冲突，HOLA 
Sync将提示错误信息，即本地a.doc文件将有红色图
标。
4.	如果这两个文件夹里有其他不存在冲突的文件，他们
将被正确同步。

8. 附录B - 用HOLA 云盘处理的特殊文件
以下列表是针对Windows系统的一些文件特殊同步处理方法的描述：

隐藏文件
隐藏文件的处理方式和普通文件一样，可以通过云盘映射、提交、
更新
临时文件
临时文件不能被云盘使用，不能对这些文件进行映射、提交、更
新或同步等设置。
快捷方式
快捷方式不能被云盘使用，它们会被云盘忽略，因此不能建立映
射、不能进行提交、更新和同步。
空白文档
空白文档和正常文档一样可以被正常处理，也可以当作空文件处
理。
