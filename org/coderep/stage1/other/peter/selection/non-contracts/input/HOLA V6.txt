第二部分: 用户管理
2.1 新建用户信息
　　方法一：
　　在导航栏选择相应的工作区（站点/组织/项目），选择“用户”，在右侧的工具栏里点击“新建”按钮。
　　在弹出的窗口中输入“登录名”，“名字”，“姓”等用户信息，然后点击“确定”按钮确认。
　　　　 
　　方法二：
在导航栏选择“组织”模块，在右边“名称”栏里输入需要搜索的组织名称，或者留空，直接点击“查找”
按钮，在搜索结果里双击选中需要新建用户的组织，在弹出窗口中选择组织，点击工具栏上的“新建”按
钮。在弹出的窗口中输入“登录名”等用户信息，然后点击“确定”按钮确认。
 
　　　　
注：站点工作区只有站点管理员登录后才能看到，站点下的“用户”包括站点下的所有用户。组织或项目工
作区里 “用户”模块，列出的用户仅属于该组织或项目。如果在组织工作区里新建用户，则该用户自动属于
“用户”模块。
2.2 批量导入/导出用户信息
     1） 首先导出用于批量导入用户的模板，点击工具栏中的‘导出用户列表’，下载该模板（注：建议先用上
述建用户的方法新建一个用户，这样导出的模板中就会有一条用户信息）
　　　 
     2） 模板中，红色字体标准的项是必填项，根据用户信息依次填入（注：按照第一步注释中的建议，仿照已
有的模板中用户信息依次填写其他用户信息）；
　　 


     3） 模板填写完毕之后，保存模板信息；点击工具栏中的‘导入用户’，在弹出窗口中，浏览选择已填写完
毕的模板‘Users.xls’，点击“附加文件”，在下一步中，选择显示出的用户，点确定即可。
　 

　 

　　 
2.3 编辑用户信息
1)	在导航栏选择相应的工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	在“登录名”、“名字”或者“姓”栏里输入需要搜索的字段，点击“查找”按钮列出符合条件的用户列
表，或者留空直接点击“查找”列出该工作区下所有的用户列表。
3)	选中需要修改信息的用户，双击或者点击工具栏上的“编辑”按钮，打开编辑用户窗口修改信息。
 


2.4 用户IP管理
管理员可以在管理端限制每个用户可访问HOLA系统的IP地址/地址段。 
1)	在导航栏选择相应的工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	在“登录名”、“名字”或者“姓”栏里输入需要搜索的字段，点击“查找”按钮列出符合条件的用户列
表，或者留空直接点击“查找”列出该工作区下所有的用户列表。
3)	选中需要修改信息的用户，双击或者点击工具栏上的“编辑”按钮，打开编辑用户窗口修改信息。
4)	选择设置地址段面板，添加该用户可以登陆HOLA系统的IP地址/地址段。
5)	选择增加打开地址段选择器。
6)	选择一个或者多个地址段，选择确定。
7)	返回到编辑用户窗口的地址段面板。 
8)	选择确定保存更改。     
标注:若打开地址段选择器为空，则没有创建IP地址段，IP地址段的创建可参考“3.8.3 创建IP地址段”

 
2.5 激活/关闭用户
1)	在导航栏选择相应的工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	点击查找按钮列出所有用户列表。 
3)	选中一个用户，选择工具条上的激活/关闭图标。
4)	默认显示的为活动用户，关闭的用户可以选择非活动显示
 
2.6 重置密码
1)	在导航栏选择相应的工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	点击查找按钮列出所有用户列表。 
3)	选中一个用户，选择工具条上的重置密码图标. 
4)	系统会自动将用户密码设置为123456。  
2.7 修改密码
1)	在导航栏选择相应的工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	点击查找按钮列出所有用户列表。 
3)	选中一个用户，选择工具条上的修改密码图标. 
4)	在新打开的修改密码窗口，输入新的密码。
5)	选择确定保存修改。
 

 
2.8 检查用户相关工作区
1)	在导航栏选择相应的工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	点击查找按钮列出所有用户列表。 
3)	选中一个用户，选择工具条上的关联工作区 图标. 在打开的窗口中可以查看该用户所属的组织和项目。
 
第三部分：组织工作区管理
3.1 新建组织
点击左侧导航栏中的‘组织’模块，在右侧的工具栏中点击‘新建组织’，输入组织名等信息，‘确定’
 
3.2 修改组织信息
1)	点击导航中的组织
2)	在右侧查询页面选择查找列出所有组织。
3)	选中一个组织，选择工具条上的编辑按钮修改组织信息。
 

3.3 添加/移除组织用户
1)	在导航栏选择相应的组织工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	点击查找，列出所有团队成员。
3)	选择工具条上的添加按钮   打开选择用户窗口，以添加组织用户。
4)	在选择用户窗口中，选择网站搜索范围。点击搜索，所有不在此组织中的用户都会列在用户/组列表区域。我
们称这些不在组织工作区的系统用户为浮动用户。
5)	选择需要添加的用户/小组，点击添加将其加入到所选的用户/组列表区域。
6)	选择确定完成添加。 
7)	欲从该组织中删除用户，则选中该用户，选择工具条上的删除按钮   将该用户从组织中移出。

 
 

3.5 添加组织管理员角色
1)	在导航栏选择相应的组织工作区，选择“用户”模块，在右侧列出用户查找界面。
2)	点击查找，列出所有团队成员。.
3)	选中需要分配/撤销角色的用户。
4)	在工具条上，选择分配/撤销角色按钮。打开分配/撤销角色窗口。
5)	在分配/撤销角色窗口中，选中该用户的名字，选择增加按钮，打开选择角色窗口。 
6)	选择组织管理员，点击确定按钮返回到分配/撤销角色窗口。
7)	在在分配/撤销角色窗口中选中组织管理员，选择增加打开工作区选择窗口。选择一个组织后，点击确定再次
回到分配/撤销角色窗口。
8)	点击确定完成角色设置。一旦设置生效，用户名称将变为粗体。
 
 
 

如果需要将角色移除： 
1)	选中需要撤销角色的用户。
2)	在工具条上，选择分配/撤销角色按钮。打开分配/撤销角色窗口。
3)	先选中具体的组织, 点击移除按钮。
4)	然后选择组织管理员，再点击移除按钮将管理员角色彻底清除。

4．	 如何在邮件中使用HOLA智能文件夹规则

    智能文件夹规则还能运用在组织或者项目的邮件处理中，发送到指定邮件地址的邮件附件会被提取到智能文件
夹中，并按照规则自动被分类、移动到指定文件夹中。

　　当用户在管理员端新建一个组织或者项目，或者编辑一个组织或者项目的时候，选择“联系信息”一栏，可
以在下方找到“智能文件夹”的选项。其中“智能邮件地址”一栏中，为用户填写的邮件帐户。以后寄往该帐户
的邮件附件，将按照用户在管理员端设置的智能文件夹规则自动分类。“负责人”的选项，则是在文档分类出错
的时候，将收到系统提醒的用户。“间隔时间”的选项，是指按照用户填写的指定时间自动检查智能文件夹，单
位为分钟。比如此处填写1，就设定为每隔一分钟将检查一次智能文件夹（这里0是一个特殊值，如果设置为0，
那么系统则不分类智能文件夹中的文件。

 
5．	HOLA智能文件夹规则如何实际运用

如例图，当用户将智能文件夹规则建立为：
条件：<DocumentType>:Name=*|<Property>密级:Code=*
目标文件夹：Paul Y Engineering Cabinet/HOLA/<DocumentType>movement:Name/<Property>密级:Code
就会将用户端（前台）符合“密级”这个条件要求的文档自动归档到Paul Y Engineering Cabinet文档柜下面
HOLA文件夹下面的子文件夹中。因为用户在参考数据中将密级的属性值设计成I，II，III，IV，V一共5个。因
此用户必须在Paul Y Engineering Cabinet文档柜的HOLA文件夹下面建立相应的I，II，III，IV，V共5个子
文件夹。如果用户在用户端文档属性中选择了密级III而没有建立III这个子文件夹时，将无法移动文档，并会
得到邮件通知此操作错误。

 


下面例图为用户端对应的文档柜，文件夹及子文件夹。如图所示：当用户选择“movement”这个文档类型，其
‘密级’的属性值选择为I时，将按照智能文件夹规则将该份文档自动移动到Paul Y Engineering Cabinet这个
文档柜下的文件夹‘HOLA’的子文件夹‘I’中；由于在下图中所示，没有建名为‘Ⅱ’的子文件夹，则如果用户
将密级选择为II时，文档将无法移动，并得到错误通知。

 

 



3.10 报告

第五部分：HOLA 系统与AD服务器的集成
5.1 介绍
　　Microsoft Active Directory 是轻量级目录访问协议(LDAP)的一种表现形式。被集成在Windows服务器当
中，是一种被广泛应用的目录环境。
 
（Active Directory 管理界面）
　　目前仍然有很多企业或者组织以Active Directory 为基础建立了文件共享及其它很多其他基于Microsoft 
Windows 的应用程序，而后又引入了HOLA系统作为企业邮件、信息协作和文档管理的平台，为了避免用户记忆
多个系统的登录帐号和密码，客户提出了把HOLA和AD服务器集成实现单点登录的需求。
5.2 HOLA与AD 服务器集成步骤
　　HOLA系统与AD服务器的集成，即可以将客户当前使用的AD服务器的用户信息、组织信息同步到HOLA系统中
并可以根据需要把认证的过程交给AD服务器来做，从而实现了用户可以使用与登录AD域相同的帐号和密码登录
到HOLA系统。集成过程分成如下步骤：
第一步：在AD服务器中创建需要同步的角色组
 
（AD服务器配置） 
　　我们首先需要创建允许HOLA系统同步的角色组hola如上图，该组中包含两个组织(hola-org1和hola-org2)和三
个用户帐号admin,user1和user2。只有位于该角色组内的信息才回被HOLA系统同步。
注：admin帐号是必须的，否则把登录认证方式修改为AD域认证以后HOLA系统的管理员帐号admin就无法使用
了。



第二步：修改配置文件配置AD服务器IP和设置管理员帐号
打开/opt/corazio/coraapp/cfg/sso-config.xml 文件，修改相关配置参数： 
<!DOCTYPE properties SYSTEM "http://java.sun.com/dtd/properties.dtd"> 
<properties> 
　　　　　　<comment>Store Sample</comment> 
　　　　　　<entry key="initialContextFactory">com.sun.jndi.ldap.LdapCtxFactory</entry> 
　　　　　　<entry key="providerUrl">ldap://192.168.1.114:3268</entry> 
　　　　　　<!--AD administrator account and validate type --> 
　　　　　　<entry key="securityPrincipal">administrator@jossoldap.net</entry> 
　　　　　　<entry key="securityCredential">123456</entry> 
　　　　　　<entry key="securityAuthentication">Simple</entry> 
　　　　　　<!-- User Tree--> 
　　　　　　<entry key="principalUidAttributeID">sAMAccountName</entry> 
　　　　　　<entry key="usersCtxDN">cn=Users,dc=jossoldap,dc=net</entry>
　　　　　　<!-- Role Tree is not validable--> 
　　　　　　<entry key="rolesCtxDN">cn=Builtin,dc=jossoldap,dc=net</entry> 
　　　　　　<entry key="uidAttributeID">member</entry> 
　　　　　　<entry key="roleAttributeID">hola</entry> 
　　　　　　<!-- validate user condition --> 
　　　　　　<entry key="credentialQueryString">uid=username,userPassword=password</entry> 
　　　　　　<!--Organizations config list--> 
　　　　　　<entry key="orgList">hola-org1,hola-org2</entry> 
</properties> 
　　以上配置文件所示的例子中，AD服务器的IP地址为192.168.1.114，AD域为jossoldap.net，AD服务器管理
员Administrator 帐号的密码为12346。正确修改这几个参数值以后，重起HOLA系统服务，就可以使用HOLA
系统管理端的AD域组织和域用户同步功能将AD服务器中的hola角色组内的所有用户(admin,user1,user2)和组织
（hola-org1,hola-org2）信息同步到HOLA系统中。
第三步：将HOLA系统用户的认证方式修改为AD域用户认证
　　用户在HOLA系统的登录界面中输入用户名和密码以后，点击登录按钮提交登录请求，HOLA应用服务器的
登录模块在接收到请求后，先判断SSOSUPPORT参数是否为TRUE，如果是TRUE，将把认证过程交给AD服务
器来完成，否则调用HOLA系统自己的认证模块。把登录认证修改为AD域认证方式只需要把
/opt/corazio/coraapp/cfg/coraziocfg.properties 配置文件内的SSOSUPPORT 参数值修改为TRUE。然后重起HOLA系
统，就可以用AD的用户名和密码登录HOLA系统了。
